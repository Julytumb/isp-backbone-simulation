# ISP Backbone Simulation

This project simulates data traffic in the BelWü-Weathermap using **ns-3**. It allows the generation of random traffic based on the predefined topology and the analysis of the load on individual links.

## Features
- **Topology Import**: Loads routers, links, and routes from CSV files in the `sources/` directory.
- **Traffic Generation**: Generates random traffic scenarios (UDP) between routers.
- **Simulation**: Conducts the simulation in ns-3, with routing calculated using the Dijkstra algorithm based on the weights in `routes.csv`.
- **Analysis**: Logs throughput (Mbps) per second for each link in the `outputs/` files.

## Project Structure
- `parser/`: Contains the C++ simulation code (`network-simulation.cc`) and auxiliary scripts.
- `sources/`: Input data (topology).
  - `routers.csv`: List of routers.
  - `links.csv`: Physical connections.
  - `routes.csv`: Routing information and weights for Dijkstra.
- `generated_sources/`: Traffic scenarios generated by the script.
- `outputs/`: Simulation results (throughput logs, routing tables, link mapping).
- `ns-3-dev/`: The ns-3 simulation framework.

## Execution

### 1. Full Workflow (Recommended)
To start the entire process (deleting old data, generating 100 traffic scenarios, and running the simulation), use the Bash script:
```bash
chmod +x run_sim.th
./run_sim.th
```

### 2. Executing Individual Components

#### Generate Traffic
If you only want to generate new traffic scenarios:
```bash
python3 parser/genfiles.py [Count]
```
*(Defaults to 1 scenario if no count is specified)*

#### Manually Start Simulation
The simulation requires traffic files to be present in `generated_sources/`.
```bash
cd ns-3-dev
./ns3 run network-simulation
```

#### Cleanup
To delete the generated traffic files or the simulation results:
```bash
python3 parser/delgenfiles.py
python3 parser/deloutputs.py
```

## Requirements
- Python 3
- ns-3 (in the `ns-3-dev` folder)
- C++ compiler (for ns-3)
